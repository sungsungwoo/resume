여러 프로젝트를 함께 운영할 계획이므로, 각 Docker 컨테이너가 직접 443을 쓰면 충돌합니다. **호스트 nginx가 443을 받아 각 컨테이너로 전달하는 리버스 프록시 구조**가 적합합니다.

```
인터넷
  ↓ 443 (HTTPS)
호스트 nginx (SSL 종료)
  ↓ 내부 HTTP
  ├─ resume.swsung72.site  →  Docker 18080
  ├─ project2.domain.com   →  Docker 18081
  └─ project3.domain.com   →  Docker 18082
```

컨테이너 내부 `nginx.conf`는 현재 그대로 두고, 호스트에서 아래 순서로 진행합니다.

---

### 1단계 — 호스트에 nginx + certbot 설치

```bash
sudo apt update
sudo apt install -y nginx certbot python3-certbot-nginx
sudo systemctl enable --now nginx
```

---

### 2단계 — 호스트 nginx 사이트 설정 생성

```bash
sudo nano /etc/nginx/sites-available/resume
```

아래 내용 붙여넣기:

```nginx
server {
    listen 80;
    server_name resume.swsung72.site;

    location / {
        proxy_pass http://127.0.0.1:18080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

저장 후 활성화:

```bash
sudo ln -s /etc/nginx/sites-available/resume /etc/nginx/sites-enabled/
sudo nginx -t          # 문법 확인
sudo systemctl reload nginx
```

---

### 3단계 — Let's Encrypt 인증서 발급 (자동으로 443 설정까지)

```bash
sudo certbot --nginx -d resume.swsung72.site
```

certbot이 자동으로 nginx 설정에 SSL 블록을 추가하고 80 → 443 리다이렉트까지 처리합니다.

완료 후 확인:

```bash
sudo nginx -t
sudo systemctl reload nginx
```

---

### 4단계 — 방화벽에 443 포트 추가

클라우드 보안 그룹에서 443 인바운드 허용 추가하고, OS 방화벽도 확인:

```bash
sudo ufw allow 443
sudo ufw status
```

---

### 5단계 — 인증서 자동 갱신 확인

```bash
sudo certbot renew --dry-run   # 갱신 테스트
```

certbot 설치 시 cron/systemd timer가 자동 등록되어 90일마다 자동 갱신됩니다.

---

### 최종 확인

```bash
curl -I https://resume.swsung72.site
```

`HTTP/2 200` 응답이 오면 완료입니다.

이후 두 번째 프로젝트를 추가할 때는 `/etc/nginx/sites-available/`에 새 파일만 만들고 `certbot --nginx -d 도메인명` 한 번 더 실행하면 됩니다.
